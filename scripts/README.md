This folder contains the TCL scripts used by the automated hardware flow.

## Files

- [generate_all_bitstreams.tcl](generate_all_bitstreams.tcl): top-level helper that sequences bitstream generation for the reconfigurable modules.
- [implement_A1_level1.tcl](implement_A1_level1.tcl): implementation TCL for the level-1 reconfigurable design.
- [project_gen_reconfig.tcl](project_gen_reconfig.tcl): template TCL that creates the Vivado project containing the reconfigurable hardware.
- [syn_reconfig.tcl](syn_reconfig.tcl): synthesis orchestration for the reconfigurable modules.

## How to use it

- Use `project_gen_reconfig.tcl` to generate the Vivado project for your reconfigurable hardware.
- The recommended workflow is to open that project in Vivado, make the hardware changes graphically, then export the project TCL and replace this file so the automated flow can reproduce the design.
- If you prefer a fully scripted flow, edit `project_gen_reconfig.tcl` directly and keep the user hardware and shell-facing interface consistent with the rest of the repo.

## Synthesis runs

- `syn_reconfig.tcl` contains the `launch_runs` and `wait_on_run` commands for the module synthesis runs.
- When you change the reconfigurable design, update those run names so they match the Vivado runs generated by your project.
- The run names in this file must correspond exactly to the module names in the generated project.

## Practical rule

- The shell-facing wrapper and address contract are fixed by the hardware template. Adapt the reconfigurable project and its synthesis run list, not the shell interface itself.
